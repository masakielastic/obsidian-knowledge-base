# note で考えて、X の記事で圧縮することにした話

これまで調査記録や学習メモは主に note に書いてきました。  
3人の対話形式にしているのは、読みやすさのためでもありますが、自分の理解不足や論点の抜けを見つけやすいからです。

ただ、続けるうちに課題も出てきました。  
記事が増えるほど探しにくくなること。  
そして、似た論点の記事が複数できやすいことです。

さらに、理解や結論が出たあとに、長い記録を短い要約文へまとめ直す作業を毎回手書きでやるのは、かなり負担が大きいと感じるようになりました。  
分量が多いので、圧縮のための文章づくりだけでも相当重いからです。

そこで最近は、一段落ついた課題を **X のタイトルつき記事** として整理して投稿するようにしています。  
このとき、要約や構成整理には **AI も使う**ことにしました。

- note は、考えを育てる場所
    
- X の記事は、論点を圧縮して残す場所
    
- AI は、その圧縮を助ける補助役
    

そんな役割分担です。

また、X の記事を **AI による投稿の置き場** として分けることで、普段の自筆の短文投稿と区別できるのも大きいです。  
自分が直接書いた文章ではないことへの違和感を、少し抑えやすくなりました。

自筆と AI 補助を混ぜて曖昧にするより、  
「ここは AI を使って整理する場所」  
と決めたほうが、自分には自然でした。

これからは、note で試行錯誤を残しつつ、  
一区切りついたテーマは X の記事で AI を使って整理していこうと思っています。
