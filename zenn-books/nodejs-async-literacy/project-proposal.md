# Node.js 教養編 企画書案

## 企画の仮タイトル

**Node.js ユーザーのための非同期 I/O 教養**  
副題案: **イベントループの下で何が起きているのか**

タイトル候補は次のような方向性も考えられる。

- **Node.js ユーザーのためのイベントループ教養**
- **Node.js から学ぶ非同期処理の土台**
- **Node.js ユーザーのための select・kqueue・epoll 入門**
- **Node.js の下にある非同期処理**
- **非同期処理を日常語で理解する Node.js 教養編**

## 企画の目的

本書の目的は、Node.js ユーザーが非同期処理を単なる `async/await` や Promise の使い方としてではなく、イベントループ、I/O 待機、OS の通知機構、ランタイムの責務分担まで含めて、日常の言葉で説明できるようになることにある。

本書は C 言語や Rust のような実装寄りの教材ではない。  
むしろその前段として、Node.js を入口にして次のことを理解するための教養書として位置づける。

- 非同期処理とは何を解決するための仕組みなのか
- イベントループは何をしているのか
- `select`、`kqueue`、`epoll` は何のためにあるのか
- libuv は何を抽象化しているのか
- Promise や `await` はどの層の話なのか
- なぜ Node.js では細かい I/O 実装を普段意識しなくてもよいのか
- その便利さの下にどのような責務が隠れているのか

## 企画の背景

非同期処理は、Node.js の実務では日常的に使われている。  
しかし、その理解はしばしば次のような断片に分かれやすい。

- Promise の書き方
- `async/await` の構文
- 「イベントループがあるらしい」という説明
- 「Node.js はシングルスレッドらしい」という説明
- 「重い処理でブロックするな」という実務上の注意

これらは重要だが、それだけでは構造が見えにくい。  
そのため、読者は次のような疑問を持っても整理しにくい。

- なぜ Node.js で多数の接続を扱えるのか
- イベントループと Worker Pool はどう違うのか
- `await` は何を待っているのか
- Node.js の便利さはどこまでが JavaScript の機能で、どこからがランタイムや OS の仕事なのか
- Linux の `epoll` や BSD 系の `kqueue` は Node.js とどう関係するのか

本書は、これらを「Node.js の下にある仕組み」という一つの地図にまとめることを狙う。

## 本書の位置づけ

本書は、非同期処理の教材シリーズの中では **教養編** にあたる。

このシリーズ全体は、大きく三層で考える。

### 第1層: 教養
Node.js を通して、非同期処理を日常の言葉で説明できるようにする。

### 第2層: 境界
便利なランタイムやライブラリが、どこまでを隠し、どこから先の責務が残るのかを整理する。

### 第3層: 実践
C 言語や Rust で、I/O 待機、イベント通知、状態管理、バッファ管理などを自分で引き受ける。

本書はこのうち、第1層を担う。  
したがって、主眼は「epoll を書けるようになること」ではなく、「epoll が必要になる文脈を理解すること」にある。

## 想定読者

想定読者は次のような Node.js ユーザーである。

- Promise や `async/await` は普段使っているが、その下の仕組みはうまく説明できない人
- イベントループについて説明を読んだことはあるが、断片的な理解にとどまっている人
- Node.js のサーバー処理やストリーム処理に触れており、もう一段下の構造を知りたい人
- 将来的に Go、Rust、C、ネットワークプログラミング、サーバー実装などに関心がある人
- 実装そのものより先に、責務分担や仕組みの地図を理解したい人

逆に、本書の主対象ではない読者も明確にしておく。

- C 言語で `epoll_wait()` を書く実装チュートリアルを求める人
- Rust の `mio` や Tokio を直接学びたい人
- Python の `asyncio` や `selectors` の使い方を中心に知りたい人
- OS の I/O 多重化 API を網羅的に比較したい人

## 読者に持ち帰ってほしいもの

読了後、読者が次のように説明できる状態を目標とする。

- Node.js の非同期処理は、JavaScript 構文だけで成り立っているわけではない
- イベントループは「順番を回すもの」であると同時に、「待機と再開の中心」でもある
- Promise は I/O 多重化そのものではなく、未来の結果や継続を扱う抽象である
- `select` は古典的な待機方式であり、`kqueue` や `epoll` はその後の高効率な方式である
- Node.js は OS ごとの差異を libuv で吸収している
- Node.js の便利さは、多くの責務を runtime 側が引き受けていることの裏返しである
- 非同期処理は「速い魔法」ではなく、「待ちで全体を止めない設計」である

## 本書で扱う範囲

本書では、次の内容を扱う。

- Node.js における非同期処理の基本的な見取り図
- ブロッキング I/O とノンブロッキング I/O の違い
- 「待つ」とは何か
- `select` の基本的な考え方
- `kqueue` と `epoll` の役割
- Node.js 前史としてのイベント駆動サーバーの文脈
- libuv の役割
- Event Loop と Worker Pool の役割分担
- readiness ベースの発想
- Promise / `async/await` / callback と、下位の I/O モデルとの関係
- `io_uring` を補講として簡潔に扱い、`epoll` と別方向の進化であることを示す
- PostgreSQL 18 の `io_uring` 採用を比較用の話題として簡潔に扱う

## 本書で扱わない範囲

本書では、次の内容は主題にしない。

- C 言語での `select` / `epoll` / `kqueue` の実装手順
- Rust の `mio` や Tokio の API 解説
- Python `asyncio` の比較
- libevent / libuv の詳細な性能比較
- BSD 系 OS と Linux の細かいシステムコール差異
- `io_uring` の API 詳解
- Windows の IOCP 深掘り
- ベンチマーク比較を中心にした性能論

扱わないものを明確にすることで、本書を「Node.js ユーザーのための教養書」に保つ。

## 本書の中核メッセージ

本書の中核メッセージは次の通りである。

### 1. Node.js は非同期処理を日常語で学ぶ入口である
Node.js の価値は、非同期処理の内部をすべて露出することではなく、実務で使われる言葉を通して、その構造を語れるようにすることにある。

### 2. 非同期処理は「速さ」より「待ち方」の設計である
非同期処理は単に高速化のテクニックではない。待機によって全体を止めないための設計である。

### 3. Promise や `await` は最上位の表現でしかない
非同期処理の全体像は、JavaScript 構文だけでは完結しない。イベントループ、libuv、OS の通知機構が土台にある。

### 4. Node.js の便利さは責務の隠蔽の上に成り立つ
Node.js は多くの複雑さを隠してくれる。そのため使いやすいが、構造を理解しないと他言語や下位層へ進みにくい。

### 5. 教養とは、責務の地図を見失わないことである
本書における教養とは、自分でイベントループを実装しなくても、OS、ランタイム、アプリケーションの責務分担を説明できる状態である。

## 本書の構成方針

本書は、次の順序で進める。

1. Node.js の日常語から出発する
2. 「待つ」とは何かを問い直す
3. `select` を古典として理解する
4. `kqueue` と `epoll` を Node.js 前史の文脈で説明する
5. libuv がそれらをどう吸収しているかを見る
6. Event Loop と Worker Pool の違いを整理する
7. Promise / `async/await` を土台の上に置き直す
8. 補講として `io_uring` を取り上げ、ready と completion の違いを示す

この順序により、Node.js の日常語から OS に近い教養までを、無理なく接続する。

## 章立て案

## 第1章 Node.js は Promise から始まったわけではない
本書の導入。  
Node.js を JavaScript の構文機能としてではなく、多数の接続や待機をどう扱うかという文脈から見直す。

## 第2章 非同期処理は「待ち方」の設計である
blocking / non-blocking を日常語で整理し、「待っている間に他の仕事へ戻る」とはどういうことかを説明する。

## 第3章 event loop とは何を回しているのか
イベントループを単なる「順番管理」ではなく、待機・再開・配布の中心として捉え直す。

## 第4章 select という古典
複数の file descriptor をまとめて待つという発想を、Node.js ユーザーにもわかる粒度で説明する。

## 第5章 kqueue と epoll の時代
Node.js 前史として、select の限界のあとに出てきた高効率なイベント通知機構を整理する。Linux では epoll、BSD/macOS 系では kqueue という形で説明する。

## 第6章 libuv は何を隠しているのか
Node.js のクロスプラットフォーム性と非同期 I/O の土台として、libuv の役割を整理する。

## 第7章 Event Loop と Worker Pool はどう違うのか
Node.js は単なる single-thread の話ではないこと、I/O と重い処理で責務がどう分かれるかを整理する。

## 第8章 Promise と async/await を地図に戻す
Promise や `await` を、イベントループや I/O 待機の上に乗る表現として再配置する。

## 第9章 補講: io_uring は何が違うのか
`epoll` の延長線というより、別方向の I/O モデルとして `io_uring` を簡潔に紹介する。

## 第10章 補講: PostgreSQL 18 から見る I/O の別世界
Node.js が主に「多数接続を止めない」ことを重視するのに対し、データベースはストレージ I/O の並列発行を重視することを示す比較章。

## 第11章 まとめ: Node.js の便利さの下にあるもの
本書全体の振り返り。  
Node.js を、日常語と実装世界のあいだをつなぐ入口として位置づけ直す。

## 本書の文体方針

- 日常語から始め、必要なところで専門用語を導入する
- いきなりシステムコール名や構造体名を並べない
- 「わかる」よりも「説明できる」を重視する
- 正確さは保ちつつ、C 言語の密度をそのまま持ち込まない
- Promise や `await` を過大評価も過小評価もしない
- Node.js の利便性を尊重しつつ、その下にある責務分担を明らかにする
- 「Node.js は魔法ではないが、便利な翻訳層を持つ」という視点を保つ

## 本書の差別化ポイント

本書は、一般的な Node.js 入門書とも、システムプログラミング入門とも異なる。

一般的な Node.js 入門書は API の使い方が中心になりやすい。  
システムプログラミング入門は OS primitive から入りがちで、日常語での共有が難しい。

本書はその中間を狙う。

- Node.js の日常語を起点にする
- しかし JavaScript の表面だけでは終わらない
- OS の待機機構まで見渡す
- ただし C の実装詳細までは踏み込まない
- 非同期処理の共通語を作ることを重視する

## この企画が次の教材につながる点

本書は単独でも成立するが、次の教材への土台にもなる。

- C 実践編では、`select` / `epoll` / `kqueue` を自分で扱う
- Rust 発展編では、`mio` や runtime を通して primitive と抽象化の関係を見る

その意味で本書は、シリーズ全体の入口であり、非同期処理の共通語を整える役割を持つ。
