## 付録B　本書で使う読み方のチェックリスト

この付録は、本書全体で繰り返し使ってきた「読み方」を、短いチェックリストとしてまとめたものです。  
本文では、UTF-8、HTTP/2、HPACK という三つの題材を扱いました。  
題材ごとに表や用語は違います。  
しかし、読解の動作はかなり揃っていました。

この付録の目的は、各仕様の細部を暗記させることではありません。  
そうではなく、生データを前にしたときに、**最初に何を確認するか** を思い出せるようにすることです。

本文を読み終えたあとで、自分でダンプや短いテストデータを見るときは、まずこの付録の問いに戻ってください。  
全部を一気に分かろうとしなくてかまいません。  
まず、どこを見て、どの表を引き、どこまでを 1単位として切るかを確かめることが大切です。

---

### B-1. 全題材共通の最初のチェック

どの題材でも、最初に次の項目を確認します。

#### 1\. これは何を 1単位として切る題材か

- UTF-8 なら 1文字
    
- HTTP/2 なら 1フレーム
    
- HPACK なら 1ヘッダー表現
    

ここが曖昧なまま読むと、途中で全部ずれます。  
まず「何を 1単位とするのか」を言えるかどうかを確認します。

#### 2\. 最初に見る場所はどこか

- UTF-8 なら先頭バイト
    
- HTTP/2 なら先頭 9バイト
    
- HPACK なら先頭ビット
    

最初の観察地点を明確にします。  
いきなり全体を眺めて雰囲気で決めないことが重要です。

#### 3\. どの表を引くべきか

- UTF-8 先頭バイト表
    
- UTF-8 特例表
    
- HTTP/2 共通ヘッダ表
    
- HTTP/2 Type 一覧
    
- HPACK static table
    

見覚えがあっても、必要なら表に戻ります。  
「たぶんこうだろう」で進めないことが大切です。

#### 4\. まだ決めてはいけないことは何か

- UTF-8 では、長さ候補が分かっても受理はまだ確定しない
    
- HTTP/2 では、Type が分かってもフレーム境界はまだ確定しない
    
- HPACK では、見覚えのある値があっても形式判定前には確定しない
    

早すぎる確定を防ぐために、この問いを入れます。

#### 5\. 確定できるところまで切ったか

途中切れや不整合や別形式が出たときに、全部読めたふりをしないための確認です。  
読める範囲だけを確定して止まることも、正しい読解です。

---

### B-2. UTF-8 用チェックリスト

UTF-8 を読むときは、次の順番で確認します。

#### 1\. 先頭バイトを見たか

- `00`–`7F` なら 1バイト文字候補
    
- `C2`–`DF` なら 2バイト文字候補
    
- `E0`–`EF` なら 3バイト文字候補
    
- `F0`–`F4` なら 4バイト文字候補
    

最初に長さ候補を決めます。  
ここではまだ受理は確定しません。

#### 2\. 必要な continuation byte の数を数えたか

- 2バイト文字候補なら後続 1バイト
    
- 3バイト文字候補なら後続 2バイト
    
- 4バイト文字候補なら後続 3バイト
    

必要な数だけ後続バイトがあるかを確認します。

#### 3\. continuation byte の範囲を確認したか

後続バイトが `80`–`BF` に入っているかを見ます。  
ここが崩れていれば受理できません。

#### 4\. 特例表が必要な先頭バイトではないか

- `E0`
    
- `ED`
    
- `F0`
    
- `F4`
    

この四つでは、2バイト目に追加条件があります。  
continuation byte の形だけで通してはいけません。

#### 5\. 途中切れではないか

長さ候補が分かっても、必要なバイト数がそろっていなければ途中切れです。  
候補と確定を混同しないようにします。

#### 6\. そこまでを 1文字として切れるか

ここまでの条件がそろって初めて、1文字として確定できます。  
確定したら、残りの先頭へ進みます。

---

### B-3. UTF-8 で止まるべき場面

次のようなときは、先へ進まずに止まります。

- 先頭バイトから必要な長さが分かったが、残りバイト数が足りない
    
- continuation byte の範囲に入らない後続バイトがある
    
- `E0`、`ED`、`F0`、`F4` の特例条件を満たさない
    
- 途中まで正常に読めたが、その先が途中切れになっている
    

このときは、「どこまで正常に読めたか」を答えます。  
全部を無理に数えないことが大切です。

---

### B-4. HTTP/2 用チェックリスト

HTTP/2 を読むときは、次の順番で確認します。

#### 1\. まず先頭 9バイトを切ったか

HTTP/2 の各フレームは、先頭 9バイトが共通ヘッダです。  
payload から先に見ないことが重要です。

#### 2\. Length を読んだか

最初の 3バイトが Length です。  
これは payload の長さであり、フレーム全体の長さではありません。

#### 3\. Type を表で確認したか

Type の値だけを見て済ませず、表で種別を確認します。  
本文で主に使ったのは DATA (`00`) と SETTINGS (`04`) です。

#### 4\. Flags と Stream Identifier を読んだか

この本では詳細な解釈までは深入りしませんでしたが、共通ヘッダの一部として存在を意識して読むことが大切です。

#### 5\. payload の範囲を Length で決めたか

手元にある残り全部を payload にしてはいけません。  
Length の値だけを payload として取ります。

#### 6\. そこまでを 1フレームとして確定したか

- 共通ヘッダ 9バイト
    
- payload Length バイト
    

これで 1フレームが確定します。  
確定したら、その次の位置へ進みます。

---

### B-5. HTTP/2 で止まるべき場面

次のようなときは、先へ進まずに止まります。

- Length が示す payload 長より実データが短い
    
- 共通ヘッダ 9バイトを切れない
    
- Length に従って payload を取ると、残りの扱いが未確定になる
    
- 連結例で、先頭フレームがまだ確定していない
    

このときは、Type が分かっていても「読めたことにする」のではなく、  
**Length に従って 1フレームが成立しているか** を優先して判断します。

---

### B-6. HPACK 用チェックリスト

HPACK を読むときは、次の順番で確認します。

#### 1\. 先頭ビットを見たか

最初にやることは、値を当てることではありません。  
まず先頭ビットを見て、何形式かを決めます。

#### 2\. Indexed Header Field かどうかを判定したか

この本で最初に扱ったのは、先頭ビットが 1 の Indexed Header Field でした。  
ここで形式が違えば、static table の読み方だけでは押し切れません。

#### 3\. index を取り出したか

Indexed Header Field なら、次に index を取り出します。  
ここでも「見覚えがある値だから」と飛ばさないことが重要です。

#### 4\. static table を引いたか

index を見たら static table に戻ります。  
値を丸暗記したつもりでも、まず表に戻る習慣を優先します。

#### 5\. そこまでを 1ヘッダー表現として確定したか

1表現ごとに復元し、残りの先頭へ進みます。  
複数並んでいても、まとめて 1つとして見ません。

#### 6\. static table だけで読める範囲を区切ったか

別形式が出たら、その位置で止まります。  
「どこまで読めたか」を明確に答えることが大切です。

---

### B-7. HPACK で止まるべき場面

次のようなときは、static table だけで先へ進まないようにします。

- 先頭ビットが 1 でなく、Indexed Header Field ではない
    
- 途中で literal 系の別形式が現れる
    
- 後ろに見覚えのある値があっても、その手前の表現境界が確定していない
    
- 「知っている index っぽい」ことだけを根拠に進みたくなった
    

このときは、「全部読めない」ではなく、  
**どこまでなら安全に読めるか** を答えます。

---

### B-8. 題材別に見た「引く表」の確認

データを前にして迷ったときは、次の対応表に戻ってください。

| 題材 | 最初に見る場所 | 主に引く表 | 1単位 |
| --- | --- | --- | --- |
| UTF-8 | 先頭バイト | 先頭バイト表、特例表 | 1文字 |
| HTTP/2 | 先頭 9バイト | 共通ヘッダ表、Type 一覧 | 1フレーム |
| HPACK | 先頭ビット | static table | 1ヘッダー表現 |

この表は、本書全体の読解フレームの要約です。  
迷ったらここに戻ってください。

---

### B-9. 危ない近道チェックリスト

本書では、何度も「筋は良いが危ない近道」を修正してきました。  
自分で読むときは、次のような考えが出てきたら一度立ち止まってください。

#### UTF-8 でありがちな近道

- 先頭バイトだけ見て確定した気になる
    
- continuation byte の形だけで通してしまう
    
- 特例表を飛ばす
    
- 途中切れでも文字数を数えてしまう
    

#### HTTP/2 でありがちな近道

- Type だけ見て済ませる
    
- payload 側から先に意味をつける
    
- Length を見ずにフレーム境界を決める
    
- 残り全部を payload とみなす
    

#### HPACK でありがちな近道

- 先頭ビットを見ずに値を当てにいく
    
- static table に戻らず丸暗記で読む
    
- 別形式が出ても無理に static で押し切る
    
- 見覚えのある値だけ拾って読めた気になる
    

この付録で一番使ってほしいのは、実はこのチェックです。  
読解は、知識を増やすことと同じくらい、**危ない進み方を止めること** が重要だからです。

---

### B-10. 「どこまで読めたか」を答えるためのチェック

本書では、「全部読めるかどうか」ではなく、「どこまで確定できたか」を何度も重視しました。  
そのため、自分で読むときも次の問いを持ってください。

- 最初の 1単位は確定したか
    
- その次の先頭位置は確定したか
    
- 読めないのは最初からか、途中からか
    
- どの表を引けば読めるはずだったか
    
- 今の段階で無理に進むべきではない理由は何か
    

この問いを持つと、

- UTF-8 の途中切れ
    
- HTTP/2 の Length 不整合
    
- HPACK の別形式混在
    

のような場面でも、落ち着いて止まれます。

---

### B-11. 最低限の復唱用フレーズ

本書の読み方を短く復唱できるように、最低限の形に圧縮しておきます。

#### UTF-8

- 先頭バイトを見る
    
- 長さ候補を決める
    
- continuation byte を確認する
    
- 必要なら特例表を見る
    
- 1文字切る
    
- 次へ進む
    

#### HTTP/2

- 先頭 9バイトを切る
    
- Length を読む
    
- Type を確認する
    
- Length の値だけ payload を取る
    
- 1フレーム切る
    
- 次へ進む
    

#### HPACK

- 先頭ビットを見る
    
- 形式を決める
    
- Indexed なら index を取る
    
- static table を引く
    
- 1表現切る
    
- 次へ進む
    

この短い復唱だけでも、かなりの場面で立て直せます。

---

### B-12. 読み始める前の最終確認

最後に、実際にダンプや短いバイト列を前にしたときの最終確認を置いておきます。

#### 開始前に確認すること

- これは UTF-8、HTTP/2、HPACK のどれか
    
- 何を 1単位として切るか
    
- 最初にどこを見るか
    
- どの表を引くか
    

#### 読みながら確認すること

- 早く決めすぎていないか
    
- 見覚えだけで進んでいないか
    
- 境界が本当に確定しているか
    
- 途中で止まるべき場面ではないか
    

#### 読み終わったあとに確認すること

- 何を根拠にそう判断したか言えるか
    
- どの表を引いたか言えるか
    
- どこまでを 1単位として切ったか言えるか
    
- どこまで確定できて、どこから先は未確定か言えるか
    

この四つが言えれば、本書の読み方はかなり身についています。

---

### B-13. 最後に

この付録は、本文の代わりではありません。  
しかし、本文を読んだあとに何度も見返す価値があります。  
なぜなら、本書の中心は個別の知識量ではなく、**読み方の手順** だったからです。

迷ったときは、難しいことをしようとしなくてかまいません。  
次の順番に戻ってください。

- まず先頭を見る
    
- 何を 1単位として切るか決める
    
- 表を引く
    
- 確定できるところまで切る
    
- 次へ進む
    
- 無理なら止まる
    

この順番に戻れるなら、本書の目的はかなり達成できています。
