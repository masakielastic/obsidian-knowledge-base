# はじめに

HTTP/2 は「速い HTTP」として語られることが多いプロトコルです。  
しかし実際に学び始めると、

- Connection / Stream / Frame の違いが曖昧
    
- ログを見ても何が起きているかわからない
    
- gRPC や HTTP/3 との関係がつかめない
    

といった壁にぶつかることがよくあります。

本書は、HTTP/2 を「仕様」ではなく  
\*\*「観察できる構造と流れ」\*\*として理解することを目的とした入門書です。

---

## この本で扱うこと

本書では、HTTP/2 を次のように分解して理解します。

- Connection / Stream / Frame の区別
    
- HEADERS / DATA を中心としたフレームの役割
    
- 通信の基本パターン
    
- curl / nghttpd のログの読み方
    
- フロー制御と HPACK の位置づけ（概要）
    
- gRPC / HTTP/3 へのつながり
    

とくに重視するのは、

👉 **「ログを見て通信の流れを説明できること」**

です。

---

## この本でやらないこと

入門に集中するため、以下は扱いません。

- HPACK の詳細な圧縮アルゴリズム
    
- フロー制御の内部仕様
    
- RFC の網羅的な解説
    
- 実装レベルの細部
    

必要になったときに深掘りできるよう、  
まずは「構造の理解」に集中します。

---

## 想定読者

- HTTP/1.1 は理解しているが HTTP/2 は曖昧な人
    
- Web エンジニア初級〜中級
    
- gRPC や HTTP/3 をこれから学びたい人
    
- プロトコルを「構造で理解したい」人
    

---

## 読み方

本書は対話形式で進みます。

- **初学者**：疑問や混乱を提示する
    
- **解説者**：構造を整理する
    
- **実装者**：ログや現実の挙動を補足する
    

という3つの視点で、HTTP/2 を分解していきます。

---

## ゴール

本書を読み終えると、次のことができるようになります。

- HTTP/2 のログを見て流れを説明できる
    
- Connection / Stream / Frame を混同しない
    
- フレームの役割を区別できる
    
- gRPC / HTTP/3 を学ぶための土台ができる
    

---

HTTP/2 は難しいプロトコルではありません。  
ただし、「整理して見る必要があるプロトコル」です。

この本では、その整理の仕方を一緒に身につけていきます。
