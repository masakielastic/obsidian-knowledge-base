# バイナリハック入門 アウトライン

## 本書の狙い

本書は、バイナリデータを 16 進数の壁としてではなく、**境界・分岐・妥当性・状態**という観点から読むための本である。

UTF-8、HTTP/2 フレーム、HPACK を題材として、次の力を身につけることを目指す。

- バイト列を見たときに、まず境界を探す発想を持つ
- 先頭情報による分岐を読めるようになる
- encode / decode の往復で構造を確認できる
- iterator / parser / validator の違いを整理できる
- state を持つ構造がなぜ難しいかを説明できる
- 自分で最小限のテストデータを作る姿勢を身につける

本書の背骨は、次の三段階である。

1. 可変長
2. 構造
3. 状態

---

## 章構成

## 第1章 バイナリデータはなぜ難しく見えるのか

### この章の役割
本書全体の導入として、なぜバイト列が難しく見えるのかを整理する。  
難しさの正体を「16進数そのもの」ではなく、「境界」「分岐」「妥当性」「状態」が見えていないことにあると定義する。

### 主な内容
- 16進数の壁とは何か
- 読みにくさの正体は何か
- 固定長と可変長の違いが最初の分岐になる
- 本書で使う4つの観点
  - 境界
  - 分岐
  - 妥当性
  - 状態
- 本書全体の進み方
  - UTF-8
  - HTTP/2 フレーム
  - HPACK

### この章の到達点
読者が「バイナリは難しい」ではなく、「どの観点がまだ見えていないのか」を意識できるようになる。

---

## 第2章 固定長と可変長

### この章の役割
バイナリ読解の最初の基礎として、固定長と可変長の違いを理解する。  
後続の UTF-8 や HTTP/2 フレームの読み方の前提を作る。

### 主な内容
- 固定長データとは何か
- 可変長データとは何か
- 固定長の読みやすさと限界
- 可変長で必要になる「先頭情報」
- 長さを先に持つ方式と、先頭ビットで分岐する方式
- 境界を見失うと何が起きるか

### この章の到達点
読者が、どこまで読むべきかを決める方法に複数の型があると理解できるようになる。

## 第3章 UTF-8 は可変長データの教材である

### この章の役割
UTF-8 を文字コードとしてではなく、可変長データの教材として読む。  
先頭バイトが後続バイトの解釈を決める、という構造を理解する。

### 主な内容
- UTF-8 を文字コードとして見るだけでは足りない理由
- 1バイト文字と複数バイト文字
- 先頭バイトと継続バイト
- どこまで読めば1文字か
- 文字をなめる iterator 的な見方
- 壊れた UTF-8 と妥当性判定
- 「読める」と「正しい」の違い

### この章の到達点
UTF-8 を通じて、可変長・先頭情報・妥当性判定の基本を説明できるようになる。

## 第4章 HTTP/2 フレームは構造化バイナリの教材である

### この章の役割
HTTP/2 フレームを、構造化バイナリの教材として扱う。  
共通ヘッダとペイロードという形で、フィールドに分解して読む感覚を身につける。

### 主な内容
- HTTP/2 フレームの全体像
- 9バイト共通ヘッダ
- Length / Type / Flags / Stream ID
- 固定長ヘッダと可変長ペイロード
- フレームを decode してフィールドに分解する
- フィールドを encode してフレームへ戻す
- UTF-8 と HTTP/2 フレームの共通点と違い

### この章の到達点
読者が、バイト列をフィールドへ分解する parser 的な見方を持てるようになる。

### 主な内容
- iterator は何をするものか

- 役割を混ぜると何が起きるか

- 役割分担と実装設計の関係

### この章の到達点
読者が、順に読むこと・構造を組み立てること・正しさを判定することを区別して説明できるようになる。

### この章の到達点

### この章の到達点

## 章配列の意図

この本の章配列は、単に話題を並べたものではない。  
理解の負荷が少ない順に、次のような階段を作っている。

1. 難しさの正体を言語化する
2. 固定長と可変長を区別する
3. UTF-8 で可変長を学ぶ
4. HTTP/2 フレームで構造を学ぶ
5. iterator / parser / validator で概念整理する
6. テストデータ自作で理解を手元化する
7. HPACK で state を導入する
8. 状態付き parser の考え方へ進む
9. HTTP/2 実装との橋を架ける
10. 他分野への応用可能性を示す

つまり本書は、**可変長 → 構造 → 状態**という順で進みながら、最後にそれを実装読解へ接続する構成になっている。
