# バイナリハック本 企画概要

## この本は何をする本か

本書は、バイナリデータを**16進数の羅列として眺めるのではなく、境界・分岐・妥当性・状態という観点から読むための本**である。

バイト列が難しく見えるのは、数字が並んでいるからではない。  
どこで区切るべきか、何を手がかりに次の解釈が決まるのか、その入力が正しいのか、前に読んだ内容が次の意味に影響するのか、という読み方の軸が見えていないからである。

本書では、この読み方の軸を次の四つに整理する。

- **境界**：どこまでが一単位なのか
- **分岐**：先頭の情報やビット列が後続の解釈をどう決めるか
- **妥当性**：読めることと、正しいことをどう区別するか
- **状態**：入力だけでは意味が決まらず、過去の履歴が現在の解釈に影響するとはどういうことか

この四つを中心に据え、UTF-8、HTTP/2 フレーム、HPACK を題材として、バイト列を読むための基本的な考え方を段階的に学ぶ。

---

## 本書の目的

本書の目的は、読者がバイナリデータを前にしたときに、単に「難しい」「16進数が多い」と感じるのではなく、**何を見れば構造が見えてくるかを言語化できるようになること**にある。

最終的には、読者が次のことをできる状態を目指す。

- バイト列を読むとき、まず境界を探すという発想を持てる
- 固定長と可変長の違いを、実際のデータ構造の違いとして説明できる
- 先頭バイトや先頭ビットによる分岐の意味を理解できる
- encode / decode の往復で構造を確認する考え方を身につけられる
- iterator / parser / validator の役割を区別して整理できる
- state を持つ表現が、なぜ一段階難しくなるのかを説明できる
- 最小限のテストデータを自作しながら理解を深める姿勢を持てる

本書は、知識を増やすだけの本ではない。  
**読むための姿勢と、切り分けるための観点を獲得する本**である。

---

## 本書の中心となる構成原理

本書の背骨は、次の三段階である。

1. **可変長**
2. **構造**
3. **状態**

この三段階に対応する教材として、次の題材を置く。

### 1. UTF-8 — 可変長データの教材

UTF-8 は、先頭バイトが後続バイトの解釈を決める可変長表現である。  
そのため、固定長の配列を読む感覚では扱えない。

ここでは主に次を学ぶ。

- 可変長とは何か
- 先頭バイトが後続の読み方を決めるとはどういうことか
- iterator と parser の違い
- 妥当な UTF-8 と、壊れた入力の違い

UTF-8 は文字コードの話であると同時に、**可変長データを読む最初の教材**でもある。

### 2. HTTP/2 フレーム — 構造化バイナリの教材

HTTP/2 フレームは、9バイトの共通ヘッダを持つ構造化バイナリである。  
UTF-8 が「どこまで読むか」を中心にした題材だとすれば、HTTP/2 フレームは「どういう項目に分解するか」を中心にした題材である。

ここでは主に次を学ぶ。

- 固定長ヘッダと可変長ペイロードの組み合わせ
- Length / Type / Flags / Stream ID の役割
- encode / decode を往復して構造を確認する方法
- バイト列をフィールドへ分解する parser 的な見方

HTTP/2 フレームは、**構造化されたバイナリを読む練習台**として非常に優れている。

### 3. HPACK — 状態付きバイナリの教材

HPACK は、HTTP/2 ヘッダ圧縮の仕組みであり、dynamic table を通じて状態を持つ。  
ここで初めて、「入力されたバイト列だけでは意味が決まらない」という種類の難しさが現れる。

ここでは主に次を学ぶ。

- 先頭ビットによる分岐
- integer 表現の読み方
- literal / indexed / table update の切り替え
- dynamic table による state の導入
- 状態付き parser をどう考えるべきか

HPACK は圧縮技法としてだけでなく、**状態を持つ表現の入口**として扱う。

---

## 本書の独自性

本書の独自性は、単に UTF-8 や HTTP/2 の解説を並べることではない。  
次の三つの整理軸を前面に出すことにある。

### 1. UTF-8 と HTTP/2 を同じ認知フレームで読む

文字コードとネットワークプロトコルは、通常は別々の分野として学ばれることが多い。  
しかし本書では、両者を「境界」「分岐」「妥当性」という共通の見方で並べる。

これにより、個別知識ではなく、**再利用可能な読み方**として理解できるようにする。

### 2. HPACK を「圧縮」より「状態付き構造」として説明する

HPACK は圧縮方式として説明されがちである。  
だが初学者がつまずく本質は、圧縮率よりも、**状態を持つことで入力の意味が固定されなくなる点**にある。

本書では、HPACK を「状態付きバイナリ構造」という観点で整理し、HTTP/2 上級編や QPACK に進む足場を作る。

### 3. iterator / parser / validator / state という整理軸を与える

本書は、個々の仕様項目を暗記する本ではない。  
何を順に読むのか、どこで分解するのか、どこで妥当性を確認するのか、どこで状態を持つのかを区別するために、

- iterator
- parser
- validator
- state

という整理軸を繰り返し使う。

これにより、読むこと・分解すること・正しさを判定すること・履歴を持つことを混同しないようにする。

## 想定読者

本書は、次のような読者を想定している。

### バイト列や16進数に苦手意識がある人

16進数の出力を見ると身構えてしまうが、避けずに理解したい人を想定する。  
本書は、数値の暗記ではなく、読む観点の整理から入る。

### UTF-8 や HTTP/2 を題材に parser 的な考え方を学びたい人

特定の言語実装に入る前に、まず構造の見方そのものを身につけたい人に向いている。

### テストデータを自作して理解したい人

既存の大きな実装を読む前に、最小限の入力を自分で作り、何が起きるかを確かめながら理解したい人を対象とする。

### HPACK や状態付き表現への入口がほしい人

HPACK や QPACK に関心はあるが、いきなり RFC や実装の全体へ飛び込むのは重いと感じる人に向けて、前提整理を提供する。

### HTTP/2 上級編へ進む前提知識を整えたい人

RST_STREAM / GOAWAY / 状態遷移 / Sans-IO のような話題に進む前に、バイト列と構造を読む土台を固めたい読者に適している。

## シリーズ内での位置づけ

本書は HTTP/2 関連シリーズの中で、次の位置に置かれる。

- **入門書の次に読める**
- **上級書の前に読む足場になる**
- **HTTP/2 を超えて使える汎用的な読解本でもある**

シリーズ全体の役割分担は、次のように整理する。

### HTTP/2 入門書 — 観察の本

入門書では、ログ観察、Connection / Stream / Frame、代表的な通信パターンなどを通じて、HTTP/2 の全体像をつかむ。  
何が起きているかを眺め、基本用語に慣れる段階である。

### バイナリハック本 — 読む力の本

本書では、観察したものを**どのような境界と分岐で読めばよいか**を学ぶ。  
UTF-8、HTTP/2 フレーム、HPACK を通じて、バイト列を読む筋道を身につける。

### HTTP/2 中級書 — 設計の本

中級書では、ライブラリ利用、非同期処理、クラス責務、レイヤー分割などを扱い、実装や API 設計を読む力を育てる。

### HTTP/2 上級書 — 成立させる力の本

上級書では、最小限の HTTP/2 クライアント、自作テストデータ、RST_STREAM / GOAWAY、状態遷移、Sans-IO、RFC 読解などを通じて、プロトコル処理を成立させる視点を扱う。

この整理において、本書はシリーズの補足ではない。  
**観察から設計と実装へ進むための、認知的な橋渡しを担当する中核の一冊**である。

---

## HTTP/2 本との関係

本書は HTTP/2 シリーズの一冊であるが、HTTP/2 の仕様解説に閉じた本ではない。  
むしろ、本書が重視するのは、HTTP/2 を題材にしながら、**より汎用的なバイナリ読解の力を育てること**である。

そのため、本書は HTTP/2 本と次のような関係を持つ。

### 入門書との関係

入門書で見たフレームやログを、「見たことがある」から「どう読めばよいかがわかる」へ進める。  
観察した断片を、境界・分岐・妥当性の観点で再整理する役割を持つ。

### 中級書との関係

中級書でライブラリ設計やクラス責務を読むとき、その内部で何が分離されているのかを理解しやすくする。  
iterator / parser / validator / state の整理は、クラス責務や API 設計の読解にもつながる。

### 上級書との関係

上級書で最小限のクライアントや自作テストデータを扱うとき、本書の内容が前提になる。  
特に、壊れた入力、境界の確認、状態付き構造の理解は、RST_STREAM / GOAWAY / Sans-IO の理解を支える。

---

## 本書で重視すること

本書では次を重視する。

- 固定長と可変長の違い
- 境界の見つけ方
- 先頭情報による分岐
- encode / decode の往復
- iterator / parser / validator / state の整理
- 自作テストデータ
- 文字コードとネットワークプロトコルの橋渡し

重要なのは、仕様項目をすべて覚えることではない。  
**どこを見れば構造が見えるかを判断できるようになること**である。

---

## 本書で深入りしすぎないこと

本書は土台作りの本であり、次の内容には深入りしすぎない。

- exploit 的な話
- RFC の全面網羅
- HPACK 実装の完全解説
- HTTP/2 上級編に属する詳細な状態遷移
- RST_STREAM / GOAWAY / Sans-IO の本格的な実装論
- QPACK や QUIC の詳細
- 特定言語でのフル実装チュートリアル

扱わないのではなく、**本書の役割を越えて拡散しない**という方針である。

---

## 読み方の中心メッセージ

本書の中心メッセージを一文で言えば、次のようになる。

> バイト列は、数値の壁としてではなく、境界・分岐・妥当性・状態の問題として読む。

UTF-8 は可変長を学ぶ入口であり、HTTP/2 フレームは構造を学ぶ入口であり、HPACK は状態を学ぶ入口である。  
この三つを通じて、読者は「バイナリを読む」という行為を、恐れる対象ではなく、切り分け可能な問題として扱えるようになる。

---

## 本書の到達点

本書を読み終えた読者には、最終的に次のような変化を期待する。

以前は、バイト列を見ると「何が書かれているかわからない」と感じていた。  
だが読後には、「まず境界を探す」「先頭の数ビットで分岐を確認する」「妥当性判定がどこにあるか考える」「この構造は状態を持つかを確認する」といった問いを自然に立てられるようになる。

そのとき、バイナリデータは完全に簡単なものにはならない。  
しかし少なくとも、**どこから手をつければよいかわからない対象ではなくなる**。

本書は、その最初の地図を与えるための一冊である。
